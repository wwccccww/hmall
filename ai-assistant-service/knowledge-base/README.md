# AI 导购助手知识库（导入百炼）

本目录为 **云汇智选 / hmall 电商导购场景** 的静态知识文档，供在 [阿里云百炼](https://bailian.console.aliyun.com/) 中创建「知识库」后上传，再绑定到智能体应用。

**文档体例：** 每个问答为 **`##` 二级标题**（`## 问：…`），问答块之间用 **`---`** 分隔，便于百炼按标题/分隔符切成多段。顶层仅保留一个 `#` 文档标题。

## 导入后只有一个切片怎么办？

百炼会把上传文件做「解析 + 分段」，若策略偏粗，**整篇 md 可能被合成一条切片**（切片列表里只显示「切片 00」）。

1. **在控制台调整分段**（名称以当前百炼界面为准）：知识库 → 该知识库 → **分段策略 / 解析设置** → 优先选 **按标题（Markdown `#`/`##`）分段**、或 **按段落/换行**，并设置合理的 **单段最大字符数**（过小会太碎，过大易整篇一条）。
2. **重新导入或「重新学习」**：改完策略后，对文档执行 **重新切片 / 重新导入**，再查看切片列表是否变为多条。
3. **仍只有一条时**：把大文件拆成 **多个小 `.md` 文件**（例如每个问答单独一个文件）再批量上传，多数情况下每个文件至少对应 1 个切片。

本仓库已用 `##` + `---` 降低「整篇一条」的概率；若控制台无「按标题分段」，以官方文档为准选用等价选项。

## 与本服务的关系

- 在根目录 `.env` 中设置 **`HM_AI_BAILIAN_APP_ID=<你的应用ID>`** 后，`ai-assistant-service` 会通过 `BailianResponsesClient` 调用百炼 **Responses API**，由百炼完成检索增强（RAG）。
- 未配置应用 ID 或调用失败时，服务会回退到 **本地 RAG**（拉取商品检索、公开券等），详见 `../AI_ASSISTANT_LOGIC.md`。
- 本知识库侧重 **平台规则、购物说明、助手能力说明**；**实时商品价格、库存、个人订单** 仍以接口与工具查询为准，请勿在文档中写死具体 SKU 价格。

## 建议导入步骤（控制台）

1. 登录百炼控制台 → **知识库** → 新建知识库（按需选择解析与分段策略）。
2. 将本目录下 **`01`～`06` 开头的 `.md` 文件** 批量导入（或打包为 zip 按控制台要求上传）。
3. 创建或打开 **智能体/应用** → 绑定上述知识库 → **发布** 应用。
4. 复制应用 ID，写入项目根目录 `.env`：`HM_AI_BAILIAN_APP_ID=<应用ID>`，并保证 `HM_AI_LLM_API_KEY` 与控制台一致。
5. 重启 `ai-assistant-service` 后，通用导购类问题将优先走百炼内置 RAG。
